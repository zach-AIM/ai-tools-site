- Anthropic announces **Claude Tag**, which they plan to eventually replace Claude Code.  
  "Tag" references the `@claude` tag used in Slack (for now; likely coming to Teams and other platforms).  

  This is a team-wide Slack integration that allows each channel to have a cloud-based Claude Code instance with the context of the channel.  
  This seems like it will be a very common use case in 6–12 months across many companies.  

  Benefits:
  - Better cost control at the team/channel level  
  - Shared learning across users interacting with the same agent  

  **Resources:**
  - Introducing Claude Tag  
    https://support.claude.com/en/articles/15594475-what-is-claude-tag  
  - Theo (YouTube): https://youtu.be/tOC2N0B9lio?is=KVH6kR4XwowF1n0f

- There’s a deeper discussion around finding the right UI/UX for AI use cases.

  - Local agents (e.g., on Mac Minis)
  - Cloud agents embedded in team chats  

  **Question:** What’s next, and which forms will stick?

## Local AI

- **GLM-5.2** (China, open weights)  
  - Benchmarks near Opus 4.8 / GPT-5.5  
  - Real-world coding performance closer to Opus 4.6 / GPT-5.4  
  - Specs: 753B params, ~1.5TB unquantized  

  Running locally:
  - Requires ~2×128GB AMD Strix Halo machines  
  - Likely impractical for most users  

- **DwarfStar (DeepSeek v4 Flash local)**  
  - Better practical performance in tests  
  - Works with 4-bit quantization (vs. GLM’s 2-bit constraint)

  **Donato Capitella (YouTube):**  
  https://youtu.be/PkKXm_mKCCM?is=nhk7Pw0R7i0WrjLk

- **Unsloth** (Australia)  
  - Open-source quantization tools + pre-quantized models  
  - GLM-5.2 support: https://unsloth.ai/docs/models/glm-5.2

- **Qwen-AgentWorld**  
  - Predictive “world model” for agents  
  - Focus: environments like terminal, SWE tasks, search, GUIs (web, Android), MCP  

  Key idea:
  - Predictive models take a state + action → predict next state  
  - Useful for training data generation rather than direct user interaction  

  **Resources:**
  - Model: Qwen/Qwen-AgentWorld-35B-A3B · Hugging Face  
  - Blog: https://qwen.ai/blog?id=qwen-agentworld  
  - Sam Wirreveen (YouTube): https://youtu.be/VzmMQWRhlBw?is=9EdUIdZEBaK8e4ml

- **Ornith 1.0 (Deep Reinforce)**  
  - Likely a fine-tuned Qwen-based model  
  - Architecture: `Qwen3_5MoeForConditionalGeneration`  

  Performance:
  - 397B model > Qwen-3.7 Max (closed), < GLM-5.2  
  - 35B model > Qwen-3.6-35B, near Qwen-3.5-397B  

  Potential upgrade candidate vs. Qwen3.6-35B-A3B in homelab setups  

  **Resources:**
  - Blog: *Ornith-1.0: Self-Scaffolding LLMs for Agentic Coding*  
  - Hugging Face:  
    - deepreinforce-ai/Ornith-1.0-397B  
    - deepreinforce-ai/Ornith-1.0-35B
